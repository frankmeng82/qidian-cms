# QA 测试提示词 - 奇点影视CMS

## 测试目标

对奇点影视CMS进行全面测试，确保系统稳定、功能完整、性能达标。

---

## 提示词 1: 单元测试

```
请为奇点影视CMS编写完整的单元测试，使用 Jest + Supertest。

测试范围：
1. 视频服务测试
   - 创建视频
   - 更新视频
   - 删除视频（软删除）
   - 获取视频列表（分页、筛选）
   - 获取视频详情

2. 用户服务测试
   - 用户注册
   - 用户登录
   - Token刷新
   - 密码加密验证

3. 采集服务测试
   - 解析采集规则
   - 数据去重逻辑
   - 图片下载

4. 工具函数测试
   - 日期格式化
   - 数据验证
   - 响应封装

要求：
1. 测试覆盖率 > 80%
2. 使用 mock 隔离外部依赖
3. 每个测试独立，不相互依赖
4. 包含正常和异常场景
5. 生成覆盖率报告
```

## 提示词 2: API 集成测试

```
请为奇点影视CMS编写 API 集成测试，使用 Jest + Supertest。

测试接口：

1. 内容接口
   - GET /api/v1/videos - 视频列表
   - GET /api/v1/videos/:id - 视频详情
   - POST /api/v1/videos - 创建视频
   - PUT /api/v1/videos/:id - 更新视频
   - DELETE /api/v1/videos/:id - 删除视频
   - GET /api/v1/categories - 分类列表
   - GET /api/v1/search - 搜索

2. 用户接口
   - POST /api/v1/auth/register - 注册
   - POST /api/v1/auth/login - 登录
   - GET /api/v1/user/profile - 用户信息
   - GET /api/v1/user/favorites - 收藏列表
   - POST /api/v1/user/favorites - 添加收藏

3. 采集接口
   - GET /api/v1/collect/rules - 规则列表
   - POST /api/v1/collect/rules - 创建规则
   - POST /api/v1/collect/execute - 执行采集

测试场景：
1. 正常请求返回 200
2. 参数错误返回 400
3. 未授权返回 401
4. 无权限返回 403
5. 资源不存在返回 404
6. 分页参数边界测试
7. 并发请求测试

要求：
1. 使用独立测试数据库
2. 每个测试后清理数据
3. 测试数据使用 factory 模式生成
```

## 提示词 4: 性能测试

```
请为奇点影视CMS编写性能测试，使用 k6 或 Artillery。

测试场景：

1. 首页加载性能
   - 并发用户：100
   - 持续时间：5分钟
   - 目标：P95响应时间 < 2秒

2. 视频列表 API 性能
   - 并发请求：1000 QPS
   - 持续时间：5分钟
   - 目标：平均响应时间 < 500ms

3. 视频播放性能
   - 并发播放：500
   - 持续时间：10分钟
   - 目标：播放成功率 > 99%

4. 搜索性能
   - 并发搜索：500
   - 持续时间：5分钟
   - 目标：P95响应时间 < 1秒

5. 采集任务性能
   - 单次采集：100条数据
   - 目标：完成时间 < 30秒

测试指标：
- 响应时间（平均/P50/P95/P99）
- 吞吐量（QPS/TPS）
- 错误率
- 并发用户数
- 系统资源（CPU/内存/数据库连接）

要求：
1. 生成性能测试报告
2. 包含性能瓶颈分析
3. 提供优化建议
```

## 提示词 5: 安全测试

```
请为奇点影视CMS进行安全测试。

测试项目：

1. SQL注入测试
   - 在输入框注入 SQL 语句
   - 验证是否被过滤或转义
   - 测试接口：搜索、登录、表单提交

2. XSS 测试
   - 在输入框注入脚本
   - 验证是否被转义
   - 测试接口：评论、视频标题、用户昵称

3. CSRF 测试
   - 验证敏感操作是否有 CSRF Token
   - 测试接口：修改密码、删除数据

4. 认证测试
   - 未登录访问需要认证的接口
   - Token 过期处理
   - 密码强度验证
   - 登录失败锁定

5. 权限测试
   - 普通用户访问管理员接口
   - 水平越权（访问其他用户数据）
   - 垂直越权（提升权限）

6. 文件上传测试
   - 上传非图片文件
   - 上传大文件
   - 上传恶意脚本
   - 路径遍历攻击

7. 敏感信息泄露
   - 检查错误信息是否暴露堆栈
   - 检查 API 响应是否包含敏感字段

要求：
1. 使用 OWASP ZAP 或手动测试
2. 生成安全测试报告
3. 对发现的问题提供修复建议
4. 复测验证修复效果
```
