# Agent 任务拆解

## 1. 角色拆分

| 角色 | 核心职责 | 需要输入 | 应输出到哪份文档 |
|---|---|---|---|
| PM / Requirements | 需求补全、范围控制、优先级 | Brief、PRD | PRD、风险清单 |
| Architect | 模块边界、技术方案、依赖关系 | Brief、PRD、流程 | 架构说明、技术决策 |
| Backend | 领域服务、接口、业务规则 | PRD、架构、数据模型 | API契约、技术设计 |
| Frontend | 页面、状态、交互、异常态 | PRD、用户流程、UI规范 | UI/UX规范、组件设计 |
| Data | 实体、字段、迁移、约束 | PRD、架构 | 数据模型、采集规则 |
| QA | 测试策略、验收用例、回归范围 | 所有功能文档 | 测试与验收 |
| Security | 权限、安全、合规 | 架构、API | 非功能需求-安全 |
| DevOps | 环境、构建、部署、回滚 | 架构、交付计划 | 交付计划、部署脚本 |

## 4. Agent 协作规则

- **缺失输入时先补文档，不能直接绕过**
- **每个角色都要明确输出物**
- **高风险项必须单独标红**
- **阶段结束后要更新 `00-SPEC-INDEX.md`**
- **每日同步进度，阻塞问题立即升级**

## 6. 风险应对

| 风险 | 应对策略 |
|---|---|
| 采集服务开发延期 | 优先保证手动添加视频功能，采集可Phase 2后期完成 |
| 播放器兼容性问题 | 使用成熟方案Video.js，预留2天测试时间 |
| 数据库性能问题 | 预留1天优化时间，准备Redis缓存方案 |
| 前端进度滞后 | 先保证管理后台，用户前台可简化 |
