# 项目 Brief

## 1. 项目一句话

打造一个现代化的影视内容管理系统，兼容苹果CMS采集生态，采用前后端分离架构，简化冗余功能，专注于核心视频管理、采集和播放体验。

## 2. 背景

- **当前现状**：苹果CMS功能臃肿，技术栈老旧（PHP），维护困难
- **为什么现在要做**：需要更现代化的技术栈（React+Node.js），更好的扩展性和维护性
- **如果不做**：继续使用老旧系统，技术债务累积，用户体验受限

## 3. 目标

- **业务目标**：建立自主可控的影视CMS平台，支持多种采集源
- **用户目标**：站长可以快速搭建影视站点，用户获得流畅的观看体验
- **技术目标**：前后端分离，容器化部署，易于扩展和维护

## 4. 范围

### 本期要做

- 视频管理（增删改查、多播放源、分集）
- 文章管理（图文内容）
- 分类标签系统
- 采集系统（兼容苹果CMS规则）
- 播放器（多线路、记忆播放）
- 用户系统（注册登录、收藏历史）
- 管理后台
- 用户前台

### 本期不做

- 专题、明星、角色、剧情模块（可后期扩展）
- 支付系统（Phase 2）
- 弹幕系统（Phase 2）
- APP客户端（Phase 3）

## 5. 目标用户

- **核心用户**：影视站长、内容运营者
- **次级用户**：普通观影用户
- **管理端**：系统管理员、内容审核员

## 6. 成功标准

- **业务指标**：支持日活1万用户，视频加载<3秒
- **体验指标**：页面首屏<2秒，播放成功率>99%
- **工程指标**：代码覆盖率>80%，部署时间<10分钟

## 7. 关键约束

- **时间**：6周完成MVP
- **预算**：开源方案，云服务成本<500元/月
- **技术栈**：React + Node.js + PostgreSQL + Redis
- **合规**：遵守当地法律法规，不存储侵权内容
- **外部依赖**：兼容苹果CMS采集规则

## 8. 项目阶段建议

- **Phase 1**（2周）：基础架构 + 视频管理 + 播放器
- **Phase 2**（2周）：用户系统 + 采集系统 + 搜索
- **Phase 3**（1周）：优化 + 测试 + 文档
- **Phase 4**（1周）：部署 + 上线 + 监控
